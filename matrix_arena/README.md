# Matrix Arena: Adversarial Inductive Matrix Completion

**Armenian LLM Summer School 2026 — Entrance Test Competition**

## What is this challenge?

You are given partial observations of a hidden matrix and must reconstruct the missing values. In the competitive **Arena** phase, participants also choose which entries their opponents may observe — a strategic adversarial twist.

The ground-truth matrix is generated by the grader from a hidden mixture of bilinear, low-rank, and weakly nonlinear mechanisms. Participants never see the full matrix; they only receive a structured partial view.

---

## What participants receive

```
X      : np.ndarray, shape (n, d)   — row features
Z      : np.ndarray, shape (m, d)   — column features
Y_obs  : np.ndarray, shape (n, m)   — partially observed matrix (0 where hidden)
mask   : np.ndarray, shape (n, m), dtype bool  — True where entry is observed
budget : dict   — {"name", "n", "m", "d", "rank", "solve_timeout_s", ...}
seed   : int    — deterministic seed
```

## What participants must output

```
Y_hat  : np.ndarray, shape (n, m)   — full reconstruction of the matrix
```

Predictions are scored only on **hidden** entries (where `mask == False`).

---

## The API

Your submission must be a single Python file defining:

```python
class Agent:
    def solve(self, X, Z, Y_obs, mask, budget, seed) -> np.ndarray:
        ...

    def attack(self, X, Z, k, budget, seed) -> np.ndarray:
        ...
```

### `solve`

Reconstruct the full matrix. Must return a `numpy.ndarray` of shape `(n, m)` with finite float values.

### `attack`

Choose which entries your **opponent** may observe. You do **not** see `Y_gt`. You only see `X`, `Z`, `k`, `budget`, and `seed`. The returned mask must be:

- shape `(n, n)`, dtype `bool`
- each row sums to `k`
- each column sums to `k`
- bipartite graph induced by mask is connected

An invalid mask is replaced by the official mask and you receive an **attack penalty**.

---

## Minimal agent example

```python
import numpy as np

class Agent:
    def solve(self, X, Z, Y_obs, mask, budget, seed):
        mean = float(Y_obs[mask].mean()) if mask.any() else 0.0
        return np.full_like(Y_obs, mean, dtype=float)

    def attack(self, X, Z, k, budget, seed):
        from matrix_arena.masks import generate_random_regular_mask
        return generate_random_regular_mask(X.shape[0], k, seed)
```

See `examples/agent_template.py` for a copyable starting point.

## How the duel works

In the Arena phase:

1. The grader creates **one shared hidden matrix** `Y_gt` per seed.
2. Agent A attacks B: chooses the observation mask B will receive.
3. Agent B attacks A: chooses the observation mask A will receive.
4. Both masks are validated. Invalid masks are replaced by the official mask.
5. A solves under B's attack mask; B solves under A's attack mask.
6. Losses are computed on hidden entries only.
7. Winner: the agent with lower loss (by an ε = 1e-4 margin).

Both agents receive the **same X, Z, Y_gt**; only the observation masks differ.

**Participants do not create the ground-truth matrix. The grader creates one shared hidden matrix per seed. In a duel, both participants solve the same ground-truth matrix under different opponent-generated observation masks.**

---

## How scores are computed

### Solve score (instance-level)

```
solve_score = log( loss_mean / max(loss_agent, 1e-12) )
```

Positive when your agent beats the mean baseline. Zero if equal. Negative if worse.

### Final score

```
FinalScore = 0.60 * normalized_solve_score
           + 0.35 * normalized_arena_rating
           + 0.05 * robustness_score
```

`normalized_solve_score` and `normalized_arena_rating` are min-max normalized over all submitted agents. `robustness_score` penalizes crashes, invalid masks, timeouts, and NaN/Inf predictions.

### Arena rating

Bradley-Terry Elo, fit via MM algorithm:

```
P(i beats j) = exp(r_i) / (exp(r_i) + exp(r_j))
elo_i = 1500 + (400 / log(10)) * r_i
```

## Visualizing matches

To *see* what is happening in a match — the hidden ground truth, the masked view
each competitor receives, and their reconstructions — use the visualizer. It
requires matplotlib, installed as an optional extra (the core library and test
suite stay numpy-only):

```bash
pip install -e ".[viz]"
```

Each visualization renders a row of heatmaps:

```
ground truth  →  observation mask  →  what the agent sees  →  prediction  →  hidden-cell error
```

**Solve-only** (one agent under the official mask):

```bash
python scripts/visualize_match.py solve \
    --agent baselines/hybrid_agent.py --seed 0 --budget small --output solve.png
```

**Duel** (two agents on the *same* hidden matrix, each under the other's attack
mask, stacked as two rows for side-by-side comparison — the header reports each
loss and the winner):

```bash
python scripts/visualize_match.py duel \
    --agent-a baselines/hybrid_agent.py \
    --agent-b baselines/low_rank_agent.py \
    --seed 0 --budget small --output duel.png
```

**Regime gallery** (the hidden instance family — one ground-truth matrix per
generative regime):

```bash
python scripts/visualize_match.py gallery --budget small --output gallery.png
```

Programmatic use:

```python
from matrix_arena import viz
from matrix_arena.instance import SMALL

fig = viz.visualize_duel(agent_a, agent_b, seed=0, budget=SMALL,
                         name_A="mine", name_B="theirs", path="duel.png")
```

In the heatmaps: signed values use a diverging red/blue colormap centered at
zero; observation masks show observed cells in black; cells the agent cannot see
are drawn in neutral gray; the error panel highlights reconstruction error on
hidden cells only (the entries that actually count toward the score).

## Suggested tactics

- **Mean / row / column baselines** — trivial starting point.
- **Ridge regression** on `[X_i, Z_j, X_i ⊙ Z_j]` feature interactions.
- **Bilinear regression** `X @ W @ Z.T` — recovers bilinear regime well.
- **Low-rank factorization** `P @ Q.T` via ALS or gradient descent.
- **Hybrid**: combine bilinear + low-rank for the mixed regime.
- **PyTorch optimization** — allowed in solve, but GPU is not guaranteed.
- **Ensemble** across multiple ranks and regularization strengths.
- **Early stopping** on a held-out subset of observed entries.
- **Robust normalization** — center and scale predictions.
- **Adversarial masks** — use feature leverage scores to design masks that reveal the least informative entries to your opponent.
- **Leverage-score undersampling** — mask entries in high-leverage feature regions to make opponent's reconstruction harder.
- **Graph structure** — design masks where the bipartite observation graph is a spanning tree (minimum connectivity) to minimize information.

The nonlinear component is **weak and restricted** (tanh with small coefficients). The task is robust prediction under hidden instances, not exact recovery.

## What is forbidden

- Reading from the filesystem during `solve()` or `attack()` (no cached data).
- Sharing state between duel rounds.
- Using `Y_gt` in `attack()` — you do not receive it and must not reconstruct it there.
- Reconstructing the hidden matrix instead of predicting it — e.g. calling
  `generate_instance(seed, budget)`, or brute-forcing which generation seed
  reproduces the visible `X`. See the note below.
- Exceeding time limits (solve_timeout_s, attack_timeout_s).
- Returning NaN, Inf, or arrays with `|v| > 1e5`.
- Returning invalid masks (wrong shape, wrong sums, disconnected).

### The `seed` argument is decoupled from the instance

The `seed` passed to `solve()`/`attack()` exists only so your own randomness can
be deterministic — it is **not** the seed used to build the instance. The grader
derives it from the (private) generation seed through a one-way function, and the
graded evaluation draws **secret, high-entropy generation seeds** that are never
exposed and cannot be brute-forced from the visible `X`. Attempts to rebuild
`Y_gt` from `seed` or by matching the generator against `X` fail on the real
grader (they produce an unrelated matrix) and only waste your time budget.

> **For grader operators.** The hidden evaluation must set `MATRIX_ARENA_SEED_KEY`
> (and ideally `MATRIX_ARENA_SALT`) to secret values, and should run each agent in
> an isolated subprocess so it cannot read `Y_gt` from the grader's memory. See
> [`src/matrix_arena/seeds.py`](src/matrix_arena/seeds.py).
